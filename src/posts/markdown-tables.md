## API 文档表格

### RESTful API 端点

| 方法 | 端点 | 描述 | 请求参数 | 响应状态码 |
|------|------|------|----------|-----------|
| GET | `/api/users` | 获取用户列表 | `page`, `limit` | 200, 401 |
| GET | `/api/users/:id` | 获取单个用户 | `id` | 200, 404 |
| POST | `/api/users` | 创建用户 | `username`, `email`, `password` | 201, 400 |
| PUT | `/api/users/:id` | 更新用户 | `id`, `username`, `email` | 200, 404 |
| DELETE | `/api/users/:id` | 删除用户 | `id` | 204, 404 |

### HTTP 状态码说明

| 状态码 | 含义 | 说明 | 常见场景 |
|--------|------|------|----------|
| 200 | OK | 请求成功 | GET 请求成功返回数据 |
| 201 | Created | 资源创建成功 | POST 请求创建新资源 |
| 204 | No Content | 请求成功但无返回内容 | DELETE 请求成功 |
| 400 | Bad Request | 请求参数错误 | 表单验证失败 |

| 403 | Forbidden | 禁止访问 | 权限不足 |

| 500 | Internal Server Error | 服务器内部错误 | 代码异常或数据库错误 |

## 表格最佳实践

### 表格设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| 简洁性 | 避免过多列和行 | 控制在 5-7 列以内 |
| 对齐 | 数字右对齐，文本左对齐 | 价格、数量右对齐 |
| 标题 | 清晰的列标题 | 使用简短描述性标题 |
| 排序 | 按逻辑顺序排列 | 时间倒序、重要性排序 |
| 格式 | 统一的格式 | 日期、货币格式一致 |

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 表格太宽 | 列数过多 | 拆分成多个表格或使用滚动 |
| 内容溢出 | 单元格内容过长 | 使用缩写或换行 |
| 难以阅读 | 缺少对齐 | 使用对齐语法 `:---`, `:---:`, `---:` |
| 移动端显示 | 响应式问题 | 使用简化版本或横向滚动 |

> **提示**: 表格是展示结构化数据的最佳方式，但要注意保持简洁和可读性。复杂数据可以考虑使用图表代替。
